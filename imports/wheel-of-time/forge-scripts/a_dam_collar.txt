# APPROACH: The equipped creature gains a {T} activated ability that Leashes a creature
# with mana value 2 or less. The Leash ability is defined via AddAbility$ on the
# Continuous static; the SubAbility SVar chain (DBLeashCounter, DBDelay, etc.) is
# defined at the card's top level so Forge can resolve them from the Equipment's
# context when the granted ability fires.
#
# LEASH MECHANIC NOTES (inherited from collar_training.txt / leash_bearer.txt):
# - COST SCALING: The card says "{2}, doubling each time you pay" (true exponential:
#   2, 4, 8, ...). Forge's Count math cannot natively double; "/Twice" multiplies
#   linearly (LEASH*2 -> 2, 4, 6, 8, ...). This is the closest DSL approximation.
# - "CAN'T BE LEASHED AGAIN UNTIL YOUR NEXT TURN": no Forge hook for when a
#   delayed trigger fires and control reverts. Omitted; players enforce manually.
# - The delayed-trigger re-arm loop (DBRearm) works as long as the leashed creature
#   stays on the battlefield; if it leaves, the trigger chain naturally ends.
#
