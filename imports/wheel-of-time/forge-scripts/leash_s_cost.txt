# Leash — confidence: low/medium. See notes below.
#
# LEASH MECHANIC: Permanent GainControl is granted immediately. A LEASH counter
# is placed on the stolen creature. A recurring delayed-trigger chain fires each
# of your upkeeps: add another LEASH counter, then offer to pay (LEASH count x 2)
# generic mana or the creature returns to its owner. If paid, the trigger re-arms
# for the next upkeep. This is the same idiom established in collar_training.txt,
# leash_bearer.txt, and leash_master.txt.
#
# COST SCALING: The card says "{2}, doubling each time you pay" (true exponential:
# 2, 4, 8, 16, ...). Forge's Count math cannot natively double; /Twice multiplies
# linearly (LEASH_count x 2 = 2, 4, 6, 8, ...). This is the same linear
# approximation used by all other Leash cards in this set. Players may wish to
# enforce the true doubling manually.
#
# "CAN'T BE LEASHED AGAIN": The clause "can't be leashed again until your next
# turn" (applied when the creature returns) has no Forge DSL hook. Omitted.
# Players must enforce manually.
#
# +1/+1 FOR LEASHED CREATURES: The card grants +1/+1 to all leashed creatures you
# control (any creature you control with at least one LEASH counter). This is
# modeled as a permanent floating Effect with a continuous static ability gated on
# the LEASH counter filter (counters_GE1_LEASH). Creating one Effect per cast is
# intentional — multiple casts stack correctly. "Duration$ Permanent" means the
# Effect never expires; since the static is gated on having LEASH counters, it is
# inert once all leashed creatures have been returned.
